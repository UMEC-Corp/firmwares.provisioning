    2. Token Lifecycle

    2. Token Lifecycle

    The current maintenance loop uses persisted `expires_in` metadata. Token refresh is not limited to `mqtt_connected == false`: the device now refreshes proactively before expiry, refreshes when token timing must be reconstructed after boot, and still refreshes when MQTT is down.

    The current maintenance loop uses persisted `expires_in` metadata. Token refresh is not limited to `mqtt_connected == false`: the device now refreshes proactively before expiry, refreshes when token timing must be reconstructed after boot, and still refreshes when MQTT is down.

    3. Recovery Escalation

    3. Recovery Escalation

    MQTT reconnect now uses explicit recovery thresholds. Normal reconnect remains the first line of defense, but repeated failures escalate into cloud-token repair and finally hard Wi-Fi/MQTT stack recovery.

    MQTT reconnect now uses explicit recovery thresholds. Normal reconnect remains the first line of defense, but repeated failures escalate into cloud-token repair and finally hard Wi-Fi/MQTT stack recovery.

    4. Route Handling
